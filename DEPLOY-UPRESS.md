# דפלוי ל-uPress — פתרון 404 מ-nginx

## למה מופיע 404?

ב-uPress **שורש האתר (Document Root)** הוא כמעט תמיד תיקיית **`public_html`** של החשבון, **לא** התיקייה שיצרת לפרויקט Git.

אם שכפלת את ה-Git לתיקייה כמו `soloblog` או `repo` — הקבצים נמצאים ב־`/public_html/soloblog/` אבל השרת מחפש את `index.html` ב־`/public_html/` → **404**.

## מה לעשות (בחר אחת מהאפשרויות)

### אפשרות 1 — שינוי שורש האתר (מומלץ)

בפאנל uPress (ניהול דומיין / אתר מתקדם):

1. מצא **Document Root** / **נתיב שורש האתר** / **תיקיית אחסון**.
2. שנה לנתיב המלא לתיקייה שבה אחרי **Pull** מופיעים `index.html` ו־`_next/`.
   - לדוגמה: `domains/solo.life/public_html/soloblog`  
   - או אם העתקת ל־`public_html`: הנתיב לתיקיית הפרויקט.

### אפשרות 2 — להעביר קבצים ל־public_html

אחרי **Pull** של ענף `deploy`:

1. פתח **מנהל קבצים**.
2. היכנס לתיקיית הפרויקט (או ל־`public_html` בתוך הפרויקט אם קיימת).
3. **העתק** את כל התוכן (כולל `_next`, `index.html`, תיקיות `post`, וכו') **לשורש** `public_html` של הדומיין (או מחק את הישן והדבק).

### אפשרות 3 — שכפול Git ישירות ל־public_html

1. במנהל הקבצים, צור תיקייה או השתמש ב־`public_html`.
2. קשר את ה-Git כך שהשכפול יהיה **בדיוק** לתיקייה שמוגדרת כשורש האתר.
3. משוך את ענף **`deploy`** (לא `main`).

## בדיקות מהירות

- אחרי Pull, חפש בקובץ מנהל הקבצים: האם קיים **`index.html`** באותה רמה שהדומיין מצביע אליה?
- פתח בדפדפן: `https://הדומיין שלך/index.html` — אם זה עובד אבל `/` לא, צריך `DirectoryIndex` (הוספנו `.htaccess` ו־`index.php`).

## ענף Git

האתר הסטטי נמצא בענף **`deploy`** בלבד.  
ענף **`main`** מכיל קוד מקור — **אין** בו `index.html` בשורש.

## תמיכה

אם אין אפשרות לשנות Document Root בפאנל — פתח כרטיס לתמיכת uPress ובקש להפנות את שורש האתר לתיקיית הפרויקט.
