# uPress — למה 404 ואיך מתקנים

## מה קרה?

בשרת נמשך רק **קוד המקור** מ־Git (`src/`, `package.json`…) — **בלי** `index.html`.  
שרת uPress **לא** מריץ `npm run build` ולא בונה Next.js.

## הפתרון בפרויקט הזה

בענף **`main`** יש תיקייה **`public_html/`** — שם נמצא **כל האתר המוכן** (HTML, `_next`, פוסטים וכו').

אחרי כל `git pull` חייבים להיות קבצים ב־`public_html/index.html`.

### שלב 1 — משיכה מ־Git

משוך את **`main`** (ברירת המחדל) ולחץ **Pull**.

### שלב 2 — שורש האתר (הכי חשוב)

השרת חייב להגיש קבצים **מתוך** `public_html` של הפרויקט, לא מהתיקייה שמעליה.

**אחת מהאפשרויות:**

1. **בפאנל uPress** — חפש **Document root** / **נתיב שורש האתר** / **תיקיית אתר** והגדר לנתיב המלא ל־  
   `.../public_html/הפרויקט-שלך/public_html`  
   (כלומר: התיקייה **`public_html` הפנימית** בתוך הפרויקט).

2. **או** — במנהל הקבצים: היכנס ל־`public_html` **שבתוך** הפרויקט, **בחר הכל**, **העתק**, ועבור ל־**שורש האחסון האמיתי** של הדומיין (ה־`public_html` הראשי של החשבון) ו**הדבק** (או החלף תוכן קודם).

### שלב 3 — בדיקה

פתח בדפדפן:

`https://הדומיין/public_html/index.html`  
או אם שינית שורש — רק `https://הדומיין/`

---

## עדכון האתר אחרי שינוי בקוד

במחשב המקומי:

```bash
npm run publish:upress
git add public_html
git commit -m "עדכון אתר סטטי"
git push
```

ואז ב־uPress שוב **Pull**.

---

## ענף `deploy`

נשאר גם ענף **`deploy`** (רק קבצים סטטיים). אפשר להשתמש בו אם מגדירים ב־Git ענף אחר — אבל עם **`public_html` על `main`** בדרך כלל מספיק למשוך `main`.
